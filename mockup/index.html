# Scenario 1: Vydanie faktúry

Majiteľ sa prihlási → klikne "Nová faktúra" → vyplní odberateľa z adresára → pridá položky → systém auto-doplní DPH → odošle emailom → PDF uloží.

# Scenario 2: Párovanie platby

Systém automaticky stiahne výpis z Tatra Banky raz denne → spáruje platby s faktúrami → nepárované označí červeno → majiteľ ich manuálne dopáruje.

# Scenario 3: Mesačný výkaz

Účtovník sa prihlási (read-only) → vyberie mesiac → stiahne PDF prehľad príjmov/výdavkov + DPH výkaz.

# Edge Cases

  • Faktúra po splatnosti → červený badge + email upozornenie
  • Duplicitná faktúra → systém varuje
  • Banka API nedostupná → retry 3x, potom notifikácia

# DESIGN.md — NEX Ledger v0.1

## 1. Architektúra systému

Trojvrstvová webová aplikácia nasadená na Ubuntu serveri cez Docker Compose.

  • Backend: Python 3.12, FastAPI, SQLAlchemy, Alembic, pg8000
  • Frontend: React 18, TypeScript, Vite, Tailwind CSS
  • Databáza: PostgreSQL 16
  • Email: SMTP cez Resend API
  • Banka: Tatra Banka REST API (OAuth2)

## 2. Databázová schéma

  • customers — id, name, ico, dic, email, address, created_at
  • invoices — id, number, customer_id, amount, vat_amount, status, issue_date, due_date, sent_at
  • invoice_items — id, invoice_id, description, quantity, unit_price, vat_rate
  • bank_transactions — id, amount, date, description, matched_invoice_id, imported_at
  • reminders — id, invoice_id, sent_at, reminder_type (first/second)
  • users — id, email, password_hash, role (owner/accountant), created_at

## 3. API Endpoints

  • POST /api/auth/login — JWT autentifikácia
  • GET  /api/invoices — zoznam s filtrom (status, date range, pagination)
  • POST /api/invoices — vytvorenie faktúry
  • PATCH /api/invoices/{id} — aktualizácia stavu
  • GET  /api/invoices/{id}/pdf — export PDF
  • POST /api/bank/sync — manuálna synchronizácia Tatra Banka
  • GET  /api/reports/monthly — mesačný výkaz (PDF + XLSX)

## 4. Frontend komponenty

  • InvoiceList — tabuľka faktúr s filtrom, sort, pagination
  • InvoiceDetail — detail faktúry, zmena stavu, PDF preview
  • InvoiceForm — formulár novej faktúry s auto-DPH výpočtom
  • CustomerSelect — combobox výberu zákazníka z adresára
  • BankTransactionList — prehľad transakcií, manuálne párovanie
  • Dashboard — prehľadové metriky (pohľadávky, splatné, mesačný obrat)

## 5. Automatizácia

  • Tatra Banka sync: každý deň o 8:00 cez APScheduler
  • Upomienky: kontrola splatnosti každý deň o 9:00 — email po 3 a 14 dňoch
  • Cleanup starých sessions: každú nedeľu

## 6. Deployment

Docker Compose — 3 services: backend (port 8000), frontend (port 3000, Nginx), postgres (port 5432 interný). Reverse proxy: Nginx na porte 80/443.

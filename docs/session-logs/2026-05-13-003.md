## Key decisions

- **L-015 + universal §10.1 CI monitoring** — proces fix nad code fix. CI failure nie je len style problem; je proces problem (push-and-forget anti-pattern). Pre-commit hook + univerzálna rule v všetkých 3 CLAUDE.md zdrojoch (project + KB common + template).

- **Hard delete > soft delete** (sémantika "trash" ikony) — UI "trash" znamená delete. Ak chceme deaktivovať, máme samostatný toggle. Soft-delete cez DELETE endpoint bol sémantický mismatch ktorý blokoval recreation cez UNIQUE constraint. Director regression case ("tibi recycle") odhalil bug okamžite.

- **Password min 5** (Director directive, internal app) — relaxed Pydantic constraint. NEX Studio nie je public-facing service, internal team usage, Director-controlled credentials. Bezpečnostne defendable v internal context. Dokumentované vo všetkých dotknutých Pydantic modeloch + frontend `PASSWORD_MIN_LENGTH`.

- **PATCH then changePassword (sequential, not atomic)** — žiadny single endpoint pre user update + password change. Sequence guard: PATCH first, password change second. Failing PATCH zachová pôvodné heslo. Pragmatic trade-off — endpoint refactor (single `PATCH /users/{id}` accepting password) by bol cleaner ale by porušoval current API contract.

- **UserForm vlastní formulárový state** (parent vlastní len in-flight + error state) — separation: UserForm = forma; parent = API integration. `UserFormData` ako contract medzi nimi. Parent rozhoduje ktoré API calls (createUserApi vs updateUserApi + changePasswordApi).

- **`key={`edit-${editingUser.id}`}`** v `<UserForm mode="edit" />` — force-remount per user pri switch medzi edit cieľmi. Bez `key` by useState s initialFromUser nereagoval na nový `initial` prop (React useState ignores subsequent initial values).

- **Director's CLEAN_CODE direktíva ako tech debt trigger** — Director si všimol duplikáciu UX-úrovni ("vyzerajú dizajnovo ináč"). To bol leading indicator pre code-level duplikáciu (~280 LOC). User experience consistency = signál pre engineering consistency.

## Tracks

1. **Bootstrap gap — forked-version DB had zero `system_settings`.** A fresh v4 fork starts with an empty `system_settings`, so `template_init_script_path` fell to its `""` default → the icc-claude-template bootstrap was silently OFF → new projects scaffolded only `docs/` (no `.claude` charter) → the AI ran charter-less → `parse_exhaustion` ("Chyba spracovania výstupu"). Fixed by seeding the setting via `system_setting.upsert`. See [[project_nex_studio_forked_version_bootstrap_gap]].

2. **Status-block parsing robustness.** The model emits its status block as BARE JSON (no `<<<PIPELINE_STATUS>>>` fence). It leaked raw JSON into `payload.report` (cockpit bubble) and, when structured_output was absent, wedged the turn with `parse_exhaustion`. Fixed: `extract_report_body` strips a bare block; `parse_status_block` recovers one (commit `2bb41a4`).

3. **Cockpit legibility (#2-adjacent).** BlockRecoveryBar no longer repeats the full question; ConversationComposer collapses to a pointer under a recovery bar (commit `2bb41a4`).

4. **Návrh re-work plan guard.** After the Auditor's Decision Cards, the re-work's plan-less navrh gate_report was rejected by a "navrh gate_report requires a plan" guard → `parse_exhaustion`. Fixed with a contextvar relaxation (`2bb41a4`) — later SUPERSEDED by track 5 (the guard was removed entirely).

5. **#1 — Task plan moved Návrh → Programovanie start.** The Vizuál keeps refining the app after Návrh, so a plan built in Návrh is stale. Now Návrh emits the design doc ONLY; the plan is generated at `_run_build_round` entry from the final design + Vizuál changes. Guards + tests reworked; the dynamic implementer sub-agent also caught 3 follow-on core regressions (fast_fix, `_fetch_cross_cutting_rules`, `_latest_navrh_gate_report_payload`). Commits `4eb4f44` + `191a7da`.

7. **#3 — one commit per Vizuál session.** The AI no longer commits each change (only writes the FE — HMR reflects it); the engine squashes the whole session into ONE commit at Vizuál approval (`_commit_vizual_changes`). Commit `b3c073c`.

## Key decisions

- **Three specialized agents** (Designer / Implementer / Auditor). Dôvod: mental clarity, fokusovaný kontext per agent, architektonicky vynútená disciplína (Designer nemá Edit/Write access do `backend/`/`frontend/`, Auditor je read-only voči kódu).

- **BE↔FE contract = OpenAPI 3.1, schema-first.** Lokácia `docs/specs/api/openapi.yaml`. Jeden súbor pre REST + SSE eventy. Schema-first (nie code-first) — Designer píše schému **pred** implementáciou, Implementer kód dodržuje schému, Auditor verifikuje runtime zhody.

- **Semantic rename DESIGN.md → ARCHITECTURE.md.** Pôvodný `DESIGN.md` obsahuje architektúru (komponenty, štruktúra), nie vizuál — meno bolo zavádzajúce. `UI_DESIGN.md` (vizuál) sa stáva novým `DESIGN.md`. Sémantika: DESIGN = ako to vyzerá, ARCHITECTURE = ako je to postavené, BEHAVIOR = ako sa to správa. BE má 2 dokumenty (ARCHITECTURE + BEHAVIOR), FE má 3 (ARCHITECTURE + BEHAVIOR + DESIGN). Konzistentné s ICC "Architect" namingom v Standing Rules.

- **Doc tree shape — variant C** (vývojová parent + sumárna executive). 9 artefaktov total v `docs/specs/`.

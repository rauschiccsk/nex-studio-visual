## Key decisions

- **(C) 1:1 port strategy** schválená Directorom — žiadne zjednodušenie, zachovaná identita s NEX Command (chunk format, score threshold 0.3, payload schema).
- **Pydantic Settings refactor** vs NEX Command bare module constants — zarovnanie s NEX Studio konvenciami pri zachovaní behaviour parity.
- **Graceful degradation** ako tzv. quality-grade pattern — Qdrant nedostupnosť nesmie blokovať disk write (a teda KB write z UI). NEX Command už takto funguje, port to len rešpektuje.
- **`project_slug` mismatch = warning, not 4xx** — UI bug catcher, nie hard validation. Reaguje na realitu, že kategórie môžu byť zložené (`projects/<slug>/<subdir>`) a strict prefix check by produkoval false positives.
- **Test fixture pattern: `_FakeReader` namiesto monkeypatch reader na úrovni mock.patch** — explicit deterministic outputs ľahšie čitatelné v RBAC tests.
- **`backend` logger propagate=False vs caplog** — pre test `test_post_project_slug_mismatch_logs_warning` `monkeypatch.setattr(backend_logger, "propagate", True)` — restored automatically. Dokumentuje subtle test-environment quirk: `main.py` disabluje propagation kvôli uvicorn access log preservation.

## Workflow disciplína + lessons

### Validation `.dedo-channel/` workflow (4 round trips Fáza 1 + 2 + 2 bug fixes)

Pattern overený robustný:
1. Dedo write do `inbox/` → Direktor relay → Implementer Read + archive
2. Implementer write plán/DONE/question → Direktor relay → Dedo Read + archive
3. Per-recipient archive (sender ostáva v inbox till recipient consumes)
4. `.dedo-channel/` gitignored — žiadny noise v commitoch

**Konvencia funguje:** žiadny copy-paste, žiadny lost context, audit trail v archive.

### Real-world smoke testing validation

Manual smoke test (Variant B z predošlej session) zachytil **2 bugy ktoré 117 unit testov nezachytili:**
1. NGINX permission bug — mocked `Path.write_text` v unit testoch nezachytil real I/O permission constraint
2. Backend port mismatch — generic template assumption ne-overený voči target projektom

**Lesson — `feedback_quality_first` v praxi:**
- Implementer's návrh Inbox Deda flag pre §10 "(d) Test Approach Verification" — real I/O testing s `tmp_path` namiesto mocking (validated by Bug #1)
- Dedo's návrh Inbox Deda flag pre Designer charter "§X.Y Pre-commit spec verification" — overiť assumptions o target projektoch cez grep/Read pred finalizáciou spec (validated by Bug #2)

Obe budú aplikované v F-006 charter updates.

### Quality-first analysis (per Director Q)

Druhá inštancia toho istého anti-patternu dnes — pri Bug #2 fix som ponúkol variant menu (A quick fix, B amendment, C disable) namiesto **jedného najlepšieho riešenia per quality-first**. Direktor flag-oval, ja prevzal Variant B priznaním inkonzistencie.

**Riešenie:** Default ponuka = jedno best option per 4 kritérií. Alternatívy LEN ak rovnocenné. Variant menu s obviously-bad options je memory anti-pattern.

### Bug fix — F-003 nginx PermissionError (commit `0e61ada`)

**Class 1 bug per Implementer charter §16** (spec OK, implementácia chyba).

- **Fix 2:** Deploy summary 3-step sudo activation pokyn Direktorovi (`sudo cp` + `sudo ln -sf` + `sudo systemctl reload`)
- **Fix 3:** Teardown analogicky — cleanup local + sudo deactivation messaging
- **Fix 4:** 3 real-I/O regression testy s `tmp_path` (per Dedo continuous improvement note: real I/O > mocking pre filesystem code)
- **Fix 5:** Robust failure handling — `deploy()` post-allocation phase wrap-utý do try/except + `release_port(slug)` pri exception (prevenuje port state leak ako sa stalo pri Direktorovom smoke teste)
- **Cleanup:** leaked port "dev" 19500 lokálne released (`.uat-ports.json` = `{}`)

### Návrh 2 — Implementer charter §10 4. dimenzia (test approach)

**Problém:** Implementer charter §10 self-PIV definuje 3 dimenzie (Spec Compliance, Field-Level Verification, Dead Code Detection), **neuvádza explicit test approach kritérium**. F-003 nginx PermissionError bug by sa zachytil v unit teste keby boli použité real I/O patterns (`tmp_path`) namiesto mocked `Path.write_text`.

**Návrh:** Pridať do §10 dimenziu **(d) Test Approach Verification**:

- Mocking OK len pre external services mimo dosahu (3rd party API, paid services)
- Mocking ZAKÁZANÉ pre operations s real-world permission/path/IO constraint ktorý mocking maskuje

**Posúdenie:** Všeobecný charakter — pattern platí pre všetky Implementer projects. Bug `0e61ada` je all-projekt relevantný.

## Workflow disciplína + chyby ktoré sa skryštalizovali

**Recurring "P-2 acceptance" anti-pattern — 2 inštancie dnes:**

1. **P-2 local-only claim** akceptovaný 8 dní bez overenia (`git remote -v` by ukázal že origin neexistuje) — 80+ commitov + git tag v0.1.0 zostalo nepushed do GitHub
2. **NEX Studio agents existence** assumovaná bez `ls .claude/agents/` — implementer/designer/auditor charters existovali od 12-Mája ale ja som proxy-implementoval všetko sám počas 8-dňového sprintu

**Director's preference (explicit):** "nemyslím si že riešením je pre teba uložiť pravidlo". Memory pravidlá pre tieto chyby existujú (`feedback_read_spec_before_paraphrasing`, `feedback_quality_first`) — len ich neaplikujem dôsledne. Reálne riešenie: **disciplína overovať pred akceptovaním** cez konkrétny tool call.

**Recipe (zapamätateľné, aplikovateľné pri každom turn):**
- Claim o filesystem → `ls`, `find`, `cat`
- Claim o agent existence → `ls .claude/agents/`
- Claim o git remote → `git remote -v`
- Claim o policy → `grep` v ICC docs + agent charters

**Plus moja False PASS chyba pri release verdikte:**

Audítor pri Re-Re-Gate G PASS verdict označil "docker compose build" + "docker compose up smoke" ako MÁGERSTAV pre-deploy gates (out-of-scope release audit). Ja som to akceptoval. Realne **release verdict bez aspoň 1 smoke testu live deploy je incomplete** — toto je rovnaký pattern ako P-2 acceptance, ale aplikovaný na audit verdict.
